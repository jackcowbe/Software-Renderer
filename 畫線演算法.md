# 畫線演算法 (Line Drawing Algorithm)

簡單說明這一張要說明的內容，在實作軟體渲染之前，我們得學會怎麼在螢幕上畫一條線，下面會逐一介紹我製作畫線演算法的過程。

##第一次嘗試 時間步進演算法

在螢幕上畫出一條線 最簡單的方法就是把X軸當成是我們的時間點，根據時間的流動畫出一條線。
<img width="324" height="561" alt="image" src="https://github.com/user-attachments/assets/1b698707-61c8-4877-b7eb-b25719626e18" />

在這張圖片裡，我們可以看到這是一條 Y=2X 的直線。我們來思考，隨著時間推進，X 會越來越大。我們必須確認在當前時間下，X 和 Y 分別在哪裡。所以用數學公式來表達這個「走格子累積」的過程：

#### ⏰ 時間的累加
當前總時間，本質上就是把每一步「微小的步伐時間」通通加總起來：

$$\text{當前總時間} = \sum_{i=1}^{\text{總步數}} \text{微小步伐時間}_i$$

#### 📍 位置的累積
而最終的像素坐標位置，就是從起點位置出發，加上每一步主軸前進時，副軸被迫跟著等比例抬高的「微小高度變動」的總和：

$$\text{最終像素位置} = \text{起點位置} + \sum_{i=1}^{\text{總步數}} (\text{微小高度變動} \times \text{主軸微小位移}_i)$$

說穿了，數學上的 $\sum$（加總符號），在我們 C++ 程式碼的肉體裡，其實就只是迴圈裡最簡單的 `+=` 連續累加而已！
