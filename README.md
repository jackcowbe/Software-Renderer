# Software-Renderer
軟體渲染的學習筆記

大家好 我今年大一 剛接觸程式設計半年 發布這篇筆記是因為我覺得圖形學的中文教學資源實在太少了 為了讓跟我一樣英文不好的學習者可以更有效率的學習 我將整理我的學習過程並記錄在這篇筆記中

##我做軟體渲染的動機

> **「只要你真心渴望看透本質，沒有什麼技術是學不起來的。」**

我並非本科生 而是一名設計系的學生 半年前因為學校的關西我接觸過遊戲引擎 那時候做了第一個非常簡單的遊戲 貪吃蛇 那時候於我而言非常困難 我花了快超過一百小時才完成他 我有一個夢想 我想製作一個在我頭腦中的世界 簡單來說就是把白日夢製作出來 然而一個簡單的貪吃蛇的遊戲邏輯就讓我非常吃力 我感到非常沮喪且失落 看著現在遊戲的內容可以做出這麼生動的人物和畫質
我花這麼多時間 才做出一個簡單的貪吃蛇 然而我不想放棄 我清楚知道電腦圖形學是我熱愛的領域 即使他學習過程非常困難且充滿坎坷 做完貪吃蛇以後在學長的引導下我去製作了一個塔防遊戲 然而最終只完成半成品。 而後我體驗到了資料結構的重要性 我去買了一本教科書 實作各種資料結構 這是今年寒假的學習內容 開學後我去修了系上的資料結構 可能因為是科系的關西 我們教得非常淺 老師甚至沒跟學生說甚麼是Big-O 甚麼是計算機組織 而後又去問問資工系的朋友 發現他們也是分開教的 然而我認為這些對於程式設計至關重要 我認為程式設計只是將我們解決問題的方法翻譯成電腦看的語言且是硬體好執行的演算法 而不是執著於代碼量和語法 後來我在沒有任何基礎的情況下學習了HLSL 我曾上網找過許多教學 然而我英文不好 且幾乎找不到資源 我只能像AI學習 一個一個單字問 過程非常的沒效率 甚至讓我懷疑自己根本無法學起來 我崩潰的大哭 那時候我真希望有個能教導我的老師 後來我製作出了一個會隨時間產生正弦的平面 看見他動起來的時候我高興到快哭出來 而後面臨學校資料結構的其中的作業 題目是製作一款簡易的遊戲 利用簡單的資料結構 然而我不想把作業丟給AI複製貼上 我想做一個對我技術邊界有突破的專案 那一次我非常的瘋狂 我在作業繳交前的一個禮拜挑戰用計算著色器做一個康威生命遊戲 雖然是在遊戲引擎的框架下實作 我成功了 我體驗到了GPU平行運算的力量 而後我去看了Real-time-Rendering-4 我想理解底層的運作方式 我理解了渲染的流水線 然而被一大堆的數學給困擾 但我並沒有放棄 我想就算我不是理組的學生 我應該可以學會他 即使我高中數學考零分 事實證明真的可以 即使我一個公式可能要花上幾天理解 後來我自己實作了各種光源和變換 但我總覺得缺少了點甚麼 彷彿沒了遊戲引擎我便無法做出任何畫面 於是我選擇來做軟體渲染 我並不喜歡看現成的教材跟著做一起 所以我會試著研發出每一個階段的演算法 打造出一個屬於我自己的軟體渲染器
